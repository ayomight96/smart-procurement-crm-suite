📘 Methodology & Validation

🤖 Model Overview

The Smart Procurement Assistant uses an Explainable Boosting Machine (EBM) model from the interpret library, wrapped in a CalibratedClassifierCV to improve probability estimates. It predicts whether a supplier's bid is optimal based on procurement KPIs and outputs explainable logit contributions per feature.

🔧 Features Used

  • Average Unit Price — lower is better
  • Average Delivery Days — shorter times preferred
  • Rating — customer satisfaction score (1–5 stars)
  • Defect Rate — percentage of returned or faulty items (lower is better)

📁 Dataset

The model was trained on a custom synthetic dataset simulating supplier bid behavior and outcomes. The dataset reflects real-world procurement dynamics with stratified splits to ensure balance.

🧪 Validation

Accuracy

92.4%

Log Loss

0.216

Based on a train/calibrate/test split with balanced sampling and stratification. Calibration improves confidence score alignment.

🔍 Explainability

Predictions include per-feature logit contributions extracted directly from the EBM model. Each decision is broken down by how much each feature pushed the model toward a recommendation or rejection. Probabilities are shown from both predict_proba and from raw logit sums for full transparency.

🌐 Architecture & API

  • 🔹 Python FastAPI microservice handles all AI prediction logic.
  • 🔹 /predict: Single supplier bid prediction with breakdown.
  • 🔹 /bulk-predict: Batch processing for bid comparisons.
  • 🔹 Served via an ASP.NET Core backend with Angular frontend.
